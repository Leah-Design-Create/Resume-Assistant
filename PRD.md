# 简历机器人 (Resume Assistant) — 产品需求文档 (PRD)

**版本**：v1.0  
**更新日期**：2025-03-10

---

## 一、产品概述

### 1.1 产品定位

简历机器人是一款基于 AI 的智能简历展示与对话助手，面向求职者与 HR 的轻量级工具。求职者上传 PDF 简历后，系统自动解析并生成结构化数据；HR 可通过分享链接访问，针对单份简历与 AI 对话，快速了解候选人背景、项目经验、技能与实习经历。

### 1.2 核心价值

| 角色 | 价值 |
|------|------|
| **求职者** | 一份简历可生成多份分享链接，分发给不同 HR；无需重复填写简历；支持 PDF 上传自动解析。 |
| **HR** | 无需下载简历，通过对话即可了解候选人；每份简历独立链接，可单独分享；可针对项目、技能、实习经历等快速提问。 |

### 1.3 目标用户

- **求职者**：需要向多家公司/HR 投递简历的候选人
- **HR / 招聘方**：需要快速了解候选人、进行初步筛选的面试官或招聘负责人

---

## 二、产品目标

1. **降低投递成本**：一次上传，多链接分发，减少重复操作
2. **提升沟通效率**：HR 通过对话快速获取候选人信息，减少来回沟通
3. **轻量化部署**：无需数据库，JSON 存储，适合个人或小规模使用；支持 Vercel + Render 免费部署

## 三、功能范围

### 3.1 功能模块总览

```
┌─────────────────────────────────────────────────────────────────┐
│                        简历机器人                                  │
├─────────────────────────────────────────────────────────────────┤
│  简历管理  │  AI 对话  │  链接分享  │  后台管理  │  头像识别  │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 简历管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 上传 PDF 简历 | 上传 PDF 文件，系统自动解析文字并调用 AI 提取结构化信息（候选人、项目经验） | P0 |
| 手动添加简历 | 通过表单填写候选人信息与项目经验，支持多项目 | P0 |
| 简历列表与切换 | 下拉选择当前简历，支持多份简历并存 | P0 |
| 删除简历 | 删除指定简历及其关联数据 | P1 |
| 下载原始 PDF | 对通过 PDF 上传的简历，支持下载原始 PDF 文件 | P2 |

**结构化数据字段**：

- **候选人 (Candidate)**：姓名、求职意向、最高学历、毕业学校、所在地、工作年限、核心项目数、交付率、技能、能力、头像
- **项目 (Project)**：id、标题、描述、标签、成果/影响、配图 URL、文档链接 (docUrl)

### 3.3 AI 对话

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 文本对话 | 用户输入问题，AI 基于当前简历内容作答 | P0 |
| 快捷提问 | 预设「项目经验」「核心技能」「公司经历」「到岗时间」等快捷按钮 | P0 |
| 项目经验展示 | 点击「项目经验」时展示全部项目卡片，含标题、描述、标签、成果、文档链接 | P0 |
| 公司经历匹配 | 用户问「公司经历」时，AI 仅根据简历原文中的「实习经历」回答，不添加概括性开场白 | P0 |
| 语音输入 | 支持语音转文字输入（浏览器 Web Speech API） | P2 |

**对话规则**：

- 项目经验：直接返回全部项目列表，不调用 AI
- 公司经历：基于简历原文中的实习经历回答，禁止添加「未披露公司名称」等推测性内容
- 其他问题：调用通义千问，以候选人为上下文回答

### 3.4 链接分享

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 复制分享链接 | 选中某份简历后，可复制带 `?resumeId=xxx` 的当前页面链接 | P0 |
| 仅看模式 | HR 通过分享链接打开时，仅能查看该简历并对话，无法切换简历、添加/删除 | P0 |
| 多链接分发 | 不同简历对应不同链接，可分别复制后分发给不同 HR | P0 |

### 3.5 后台管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 后台入口 | 访问 `{后端地址}/admin` 进入管理页 | P0 |
| 查看简历列表 | 展示所有简历（姓名、求职意向、ID） | P0 |
| 删除简历 | 在后台删除指定简历 | P0 |
| 上传 PDF | 在后台直接上传 PDF，识别后入库 | P0 |

### 3.6 头像识别

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 前端人脸检测 | 上传 PDF 时，用 face-api.js 在第一页检测人脸框，将坐标发给后端裁剪 | P1 |
| 服务端 VL 兜底 | 若前端未检测到人脸，则用通义千问 VL 识别人脸中心并裁剪 | P1 |
| 手动上传头像 | 支持单独上传头像图片，覆盖候选人头像 | P2 |

## 四、用户流程

### 4.1 求职者流程

```
上传 PDF / 手动添加简历 → 选择简历 → 复制分享链接 → 发给 HR
                    ↓
               （可选）上传头像、编辑项目 docUrl
```

### 4.2 HR 流程

```
收到分享链接 → 打开页面（仅看模式）→ 选择快捷提问或输入问题 → 查看 AI 回复与项目卡片
```

### 4.3 数据流

```
PDF 上传 → 文字提取 (pdf-parse) → AI 结构化 (通义千问) → 存入 data/resumes.json
                                                          ↓
用户提问 → /api/chat → 匹配项目/公司经历 或 调用 AI → 返回内容 + 可选 projects
```

## 五、非功能需求

### 5.1 性能

- PDF 上传解析：单次 < 30 秒（依赖 AI 接口）
- 对话响应：< 10 秒（依赖通义千问）
- 前端首屏：< 3 秒

### 5.2 兼容性

- 浏览器：Chrome、Edge、Safari、Firefox 等现代浏览器
- 移动端：响应式布局，支持手机访问

### 5.3 部署

- 支持本地开发：`npm run dev:all`
- 支持 Vercel（前端）+ Render（后端）免费部署
- 数据存储：`data/resumes.json`、`data/docUrls.json`、`data/uploads/*.pdf`

### 5.4 安全

- API Key 仅存于服务端环境变量，不暴露给前端
- CORS 限制为指定前端域名
- 上传文件大小限制：10MB

## 六、技术架构

### 6.1 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | React 19、Vite、Tailwind CSS、MUI、Motion、face-api.js |
| 后端 | Node.js、Express、TypeScript |
| AI | 阿里云百炼 DashScope（通义千问文本 + 视觉模型） |
| 简历解析 | pdf-parse、pdfjs-dist |
| 图像处理 | sharp、face-api.js |

### 6.2 数据模型

**Resume**

```ts
{
  id: string;
  candidate: Candidate;
  projects: Project[];
  rawText?: string;   // PDF 识别原文，用于公司经历回答
  pageImages?: string[];
}
```

**Candidate**

```ts
{
  name, title, highestDegree?, degreeSchool?, location,
  experienceYears, coreProjects, deliveryRate,
  skills: string[], capabilities: { name, level }[],
  avatarUrl: string
}
```

**Project**

```ts
{
  id, title, description, tags: string[],
  impact, imageUrl, docUrl  // docUrl 支持 GitHub 等外链
}
```

### 6.3 关键设计

- **docUrl 持久化**：项目文档链接存于 `data/docUrls.json`，按候选人+项目标题索引；删除简历后重新上传，可自动恢复已有 docUrl
- **读取时从文件加载**：每次 API 读取简历时从 `resumes.json` 重新加载，编辑文件后刷新即可生效，无需重启服务

## 七、API 概览

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /admin | 后台管理页 |
| GET | /api/resumes | 简历列表 |
| GET | /api/resumes/:id | 简历详情 |
| POST | /api/resumes | 新增简历（手动） |
| POST | /api/resumes/upload | 上传 PDF 简历 |
| DELETE | /api/resumes/:id | 删除简历 |
| GET | /api/resumes/:id/download | 下载原始 PDF |
| POST | /api/resumes/:id/avatar | 更新头像 |
| POST | /api/chat | 发送聊天消息 |
