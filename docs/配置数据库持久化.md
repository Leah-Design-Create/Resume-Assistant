# 配置数据库持久化 — 详细步骤

配置后，简历和 PDF 会存在 PostgreSQL 里，**Render 休眠或重启后数据不会丢**，别人打开你的分享链接会一直看到你的简历，不会变回「张三」。

---

## 前提

- 你已经按 [DEPLOY.md](../DEPLOY.md) 把**后端**部署到了 Render（Web Service 已创建并能访问）。
- 你的代码已经包含数据库方案（有 `server/db.ts`、`server/store-db.ts` 等），并已推送到 GitHub，Render 已用最新代码部署。

---

## 第一步：在 Render 里创建一个 PostgreSQL 数据库

1. 打开浏览器，登录 [https://dashboard.render.com](https://dashboard.render.com)。
2. 在左侧或顶部找到 **「New」** 按钮，点一下。
3. 在菜单里选择 **「PostgreSQL」**（不要选 Web Service）。
4. 进入创建数据库的页面后，按下面填写：
   - **Name**：随便起一个英文名，例如 `resume-bot-db`（只用于在 Render 里区分）。
   - **Database**：可以保持默认，或改成 `resume_db`。
   - **User**：保持默认即可。
   - **Region**：**尽量选和你后端 Web Service 同一个区域**（例如后端在 Singapore，数据库也选 Singapore），这样用 Internal URL 更稳定。
   - **Plan**：选 **Free**。
5. 点 **「Create Database」**，等一两分钟，直到状态变成 **Available**（绿色）。

---

## 第二步：复制数据库连接地址（DATABASE_URL）

1. 数据库创建好后，会**自动进入该数据库的详情页**（或从 Dashboard 左侧点你的数据库名字进入）。
2. 在详情页里找 **「Connections」** 或 **「Connect」** 这一块（有的版本在顶部 Tab 里，有的直接在页面中间）：
   - 会看到两行或两个框：
     - **Internal Database URL**（或 **Internal Connection String**）  
       格式类似：`postgresql://用户名:密码@xxx.xxx.render.com/数据库名?sslmode=require`  
       同一账号、同一区域的后端用这个，**推荐选它**。
     - **External Database URL**（或 **External Connection String**）  
       从外网也能连，Internal 连不上时再用。
3. 在 **Internal Database URL** 那一行：
   - 有的界面右侧有 **复制图标（两个重叠的方框）**，点一下即可复制整段；
   - 没有复制按钮的话，用鼠标**拖选整段 URL**（从 `postgresql://` 到末尾的 `?sslmode=require`），右键复制或 Ctrl+C。
4. 把复制好的内容粘贴到记事本里检查一下：**必须是一整行**，以 `postgresql://` 开头，中间有密码，结尾通常是 `?sslmode=require`。**不要发给别人**（里面包含密码）。

> **如果页面上没看到 "Internal"**：有的 Render 版本只显示一个 **「Connection string」** 或 **「Database URL」**，没有区分 Internal/External，用那个即可。若有两个，优先用带 **Internal** 的。

---

## 第三步：把 DATABASE_URL 填到后端 Web Service 的环境变量里

1. 在 Render 左侧或 **Dashboard** 里，找到你的**后端服务**（Web Service，名字像 `resume-bot-api`），点进去。
2. 在顶部菜单里点 **「Environment」**（环境变量）。
3. 在 **「Environment Variables」** 区域点 **「Add Environment Variable」** 或 **「+ Add」**。
4. 填写：
   - **Key**（键）：输入  
     `DATABASE_URL`  
     （必须一模一样，全大写，下划线）。
   - **Value**（值）：把第二步复制的 **Internal Database URL** 整段粘贴进去（不要前后加空格或引号）。
5. 点 **「Save Changes」** 保存。
6. 保存后 Render 会**自动重新部署**你的后端。等部署状态变成 **Live**（绿色）即可，大约 1～3 分钟。

---

## 第四步：确认是否生效

1. 打开你的**前端页面**（Vercel 的地址，例如 `https://xxx.vercel.app`）。
2. 若之前已经有简历：
   - 若**没配置数据库前**就在用，当时的数据在「文件」里，**不会自动进数据库**。  
   - 所以配置好数据库后，**需要重新上传一次你的 PDF 简历**（或在前端/后台再「添加」一次简历），这样新数据才会写入 PostgreSQL。
3. 重新上传或添加简历后：
   - 选你这份简历，点 **「复制链接」**，把链接发给自己或别人。
   - 过一段时间（例如几小时后，或第二天）再打开这个链接，**应该仍然是你这份简历**，不会变成张三。

---

## 常见问题

### 1. 配置了 DATABASE_URL 后，之前的简历不见了？

- 之前的数据是存在 Render 的「临时磁盘」里的，没有数据库时重启就清空了。
- 配置数据库后，**新上传/新添加的简历**才会进数据库。请在前端或后台（`后端地址/admin`）重新上传一次 PDF 或手动添加简历。

### 2. 该用 Internal 还是 External Database URL？

- **同一区域**（例如后端和数据库都在 Singapore）：优先用 **Internal Database URL**。
- 若后端和数据库不在同一区域，或 Internal 连不上，再改用 **External Database URL**，并确保 Web Service 能访问外网。

### 3. 保存环境变量后要自己点「部署」吗？

- 不用。保存 **Environment** 后，Render 会自动触发一次重新部署，等 **Live** 即可。

### 4. 本地开发也想用数据库可以吗？

- 可以。在项目根目录的 `.env` 里加一行：  
  `DATABASE_URL=你的 Postgres 连接地址`  
  本地用的话，可以填 Render 的 **External Database URL**（注意不要把这个 .env 提交到 GitHub）。

### 5. 免费 PostgreSQL 够用吗？

- Render 免费 PostgreSQL 有容量和连接数限制，对个人、少量简历和 HR 访问完全够用。若以后简历很多，再考虑升级。

### 6. 想直接看数据库里的数据可以吗？

- **通过应用**：打开 **后台**（`后端地址/admin`）可以看简历列表；打开**前端**选简历可以看完整内容和对话。这些读的都是数据库里的数据。
- **直接连数据库**：在 Render 数据库详情页复制 **External Database URL**，用本机的 **TablePlus、DBeaver、pgAdmin** 等工具连接（填主机、端口、数据库名、用户名、密码）。连接后可以看到表 `resumes`（简历）和 `resume_pdfs`（PDF 二进制）。Render 若提供 **Shell**，也可在网页里用 `psql` 执行 `SELECT * FROM resumes;` 查看。

---

## 小结

| 步骤 | 做什么 |
|------|--------|
| 1 | Render → New → PostgreSQL → 创建免费数据库（Region 选和后端一致） |
| 2 | 在数据库详情页复制 **Internal Database URL** |
| 3 | 后端 Web Service → Environment → 添加 `DATABASE_URL` = 粘贴刚才的 URL → 保存 |
| 4 | 等自动部署完成，重新上传/添加一次简历，再复制链接给别人 |

做完以上步骤，你的简历和 PDF 就会持久化在数据库里，分享链接不会再变回张三。
